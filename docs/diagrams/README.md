# Diagram Index

This folder is organized so each use case has its own numbered subfolder. That makes the sequence, object, and state diagrams easier to review in the same order used by the design document.

## Shared Design Diagrams

- `domain/`
  Domain model source.
- `use-case/`
  Use case overview source.
- `activity/`
  Activity diagram sources for all documented use cases.
- `architecture/`
  Package, component, and deployment diagram sources.
- `class/`
  Class diagram source.

## Use Case Diagram Folders

- `use-cases/01-schedule-appointment/`
  Sequence, object, and state diagrams for Schedule Appointment.
- `use-cases/02-receive-reminders/`
  Sequence, object, and state diagrams for Receive Reminders.
- `use-cases/03-view-dashboard/`
  Sequence and object diagrams for View Dashboard.
- `use-cases/04-update-pricing/`
  Sequence and object diagrams for Update Pricing.
- `use-cases/05-update-job-status/`
  Sequence, object, and state diagrams for Update Job Status.
- `use-cases/06-send-completion-message/`
  Sequence and object diagrams for Send Completion Message.
- `use-cases/07-send-proactive-notification/`
  Sequence and object diagrams for Send Proactive Notification.
- `use-cases/08-request-quotation/`
  Sequence and object diagrams for Request Quotation.
- `use-cases/09-check-order-status/`
  Sequence and object diagrams for Check Order Status.
- `use-cases/10-retrieve-booking/`
  Sequence and object diagrams for Retrieve Booking.
- `use-cases/11-cancel-booking/`
  Sequence and object diagrams for Cancel Booking.

State diagrams are only included where the implemented flow has a meaningful lifecycle to model.
